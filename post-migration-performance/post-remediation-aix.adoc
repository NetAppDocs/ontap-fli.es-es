Tras la migración, los hosts AIX requieren pasos de remediación para garantizar que los LUN recién importados se reconocen y funcionan correctamente.



== Post-migración - FLI offline

Para FLI sin conexión, pon el LUN en línea después de eliminar la importación de lun y confirmar que los iniciadores de host están conectados.

NOTE: Consulta la documentación del proveedor del sistema operativo para ver la lista de comandos más reciente y actualizada.

.Pasos
. Vuelve a escanear después de separar para confirmar el estado limpio.
+
[source, cli]
----
cfgmgr
----
. Identifica los dispositivos de almacenamiento.
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. Verifica que el dispositivo está conectado.
+
[source, cli]
----
lspv |grep hdiskx
----
. Identifica el grupo de volúmenes y los sistemas de ficheros.
+
[source, cli]
----
lspv
lsvg <vg_name>
lsvg -l <vg_name>
lsvg -p <vs_name>
----
. Varyonvg y monta sistemas de archivos.
+
[source, cli]
----
varyonvg <vg_name>
mount /mount_point
df -g
----




== Post-migración - FLI en línea

Después de eliminar la relación de importación, confirma que el nuevo LUN funciona de forma independiente con UNMAP activado. Después de eliminar la importación de LUN, verifica si el LUN está configurado con `-space-reserve disabled` y `-space-allocation enabled`. Si no, modifícalo en consecuencia.

[cols="4"]
|===
| Sistema de archivos | Dispositivo de bloques | LVM (FS) | Sanboot 


| Paso 1: volver a explorar el almacenamiento en todos los hosts AIX asignados | Paso 1: Vuelve a explorar el almacenamiento en todos los hosts AIX asignados. | Paso 1: volver a explorar el almacenamiento en todos los hosts AIX asignados | Paso 1: volver a explorar el almacenamiento en todos los hosts AIX asignados 


|  |  | SCSI UNMAP se activa automáticamente si el dispositivo está thin provisioned. | SCSI UNMAP se activa automáticamente si el dispositivo está thin provisioned. 
|===

. Vuelve a escanear después de separar para confirmar el estado limpio.
+

lsdev -Cc disk

+
[source, cli]

lvmstat -v vg_name -r
----
