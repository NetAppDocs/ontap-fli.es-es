.Puntos clave que debes conocer
* La configuración *ATS-only* se almacena en la cabecera VMFS, que reside en la primera extensión del almacén de datos. Si el almacén de datos abarca varias extensiones, solo es necesario corregir la primera extensión.
* Si el almacén de datos se comparte entre varios hosts ESXi, normalmente basta con actualizarlo desde un host; los demás hosts detectan el cambio después de un nuevo escaneo/montaje.
* No se puede ejecutar ninguna E/S activa en el almacén de datos mientras se cambia el modo de sólo ATS. VMware recomienda evacuar/migrar o apagar las máquinas virtuales y desmontar/volver a montar según sea necesario para aplicar el cambio.




=== Ejemplo de comandos de ESX: sync block

Poner el host en modo de mantenimiento::


esxcli system maintenanceMode set --enable true

Identifica dispositivos de almacenamiento y almacenes de datos::


[Source]
----
esxcli storage core device list
esxcli storage filesystem list
----
Comprobar el modo de bloqueo VMFS ATS::


vmkfstools --queryfs -h /vmfs/volumes/<DATASTORE_NAME>

Desmonta el almacén de datos (repítelo en todos los hosts ESXi si es compartido)::


esxcli storage filesystem unmount -l <DATASTORE_NAME>

Desconecta el dispositivo de respaldo (NMP / dispositivo naa)::


esxcli storage core device set -d <naa.ID> --state=off

Verifica que el dispositivo está desconectado::


esxcli storage core device list -d <naa.ID>

Vuelve a escanear después de separar para confirmar el estado limpio::


esxcli storage core adapter rescan --all

Salir del modo de mantenimiento después de completar::


esxcli system maintenanceMode set --enable false



== Premigración - FLI en línea

A continuación se detallan los pasos específicos del FLI online, pero estos pasos deben ir precedidos de los pasos en el FLI offline previo a la migración.

Los siguientes pasos deben llevarse a cabo después de que el NetApp LUN se aprovisione al Host.

En este punto la importación lun ya está creada, la lun está asignada al igroup del host y hay un iniciador iSCSI o FC conectado desde el host.

En caso de FLI offline, el LUN NetApp permanece en estado offline.

Poner el host en modo de mantenimiento::


esxcli system maintenanceMode set --enable true

Identifica dispositivos de almacenamiento y almacenes de datos::


[Source]
----
esxcli storage core device list
esxcli storage filesystem list
----
Comprobar el modo de bloqueo VMFS ATS::


vmkfstools --queryfs -h /vmfs/volumes/<DATASTORE_NAME>

Desmonta el almacén de datos (repítelo en todos los hosts ESXi si es compartido)::


esxcli storage filesystem unmount -l <DATASTORE_NAME>

Si se detecta solo ATS, convierte a ATS+SCSI::


vmkfstools --configATSOnly 0 /vmfs/volumes/<DATASTORE_NAME>

Desconecta el dispositivo de respaldo (NMP / dispositivo naa)::


esxcli storage core device set -d <naa.ID> --state=off

esxcli storage core device list -d <naa.ID>

esxcli storage core adapter rescan --all
